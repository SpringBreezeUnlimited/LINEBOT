# 外部仕様書

## 1. 外部インターフェース一覧
- LINE Messaging API Webhook
- Web 管理画面
- CSV ダウンロード
- GitHub Actions / 外部呼び出し用タスク URL

## 2. LINE Webhook
### 2.1 エンドポイント
- `POST /callback`

### 2.2 入力
- `X-Line-Signature` ヘッダー
- LINE から送られるイベント JSON

### 2.3 挙動
- 署名検証に失敗した場合は 400 を返す
- レート制限を超える場合は 429 を返す
- テキストメッセージのみ処理する

### 2.4 利用メッセージ
- `予約`
- `予約 <種類名>`
- `キャンセル`
- `到着`

### 2.5 返信例
- 予約完了番号と待ち人数
- 種類未指定時の案内
- 受付停止時の案内
- 呼出時の案内
- 到着受付完了の案内

## 3. 管理画面
### 3.1 ログイン
- `GET /login`
- `POST /login`

### 3.2 管理操作
- `GET /admin`
- `GET /admin/data`
- `GET /admin/type_counts`
- `POST /admin/toggle-accepting`
- `POST /admin/auto-call-count`
- `POST /admin/call/<res_id>`
- `POST /admin/finish/<res_id>`
- `GET /admin/types`
- `POST /admin/types`
- `POST /admin/types/delete/<type_id>`
- `POST /admin/types/toggle/<type_id>`
- `GET /admin/history`
- `GET /admin/history/export.csv`
- `GET /admin/login-logs`
- `POST /logout`

### 3.3 認証
- 管理画面は管理者認証が必要
- ログ閲覧は監査管理者認証が必要

### 3.4 セキュリティ要件
- POST 系は CSRF トークン必須
- 認証失敗はリダイレクトまたは 401/403 を返す

## 4. 自動呼出タスク
### 4.1 エンドポイント
- `POST /tasks/process-call-queue`

### 4.2 認証
- `BATCH_CALL_RUNNER_TOKEN` を Bearer または `X-Task-Token` で検証する

### 4.3 挙動
- 設定人数分の待機予約を呼出する
- 結果を JSON で返す

## 5. レスポンス方針
- HTML 画面はブラウザ向け
- `/admin/data`、`/admin/type_counts`、`/tasks/process-call-queue` は JSON を返す
- `export.csv` は CSV を返す

## 6. ブラウザ側通信
- 管理画面は `/admin/data` をポーリングまたは再取得に使用する
- 履歴画面は初期表示中心で、画面内の並べ替えとフィルタはフロントで再描画する
