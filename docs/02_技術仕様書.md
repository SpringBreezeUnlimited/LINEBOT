# 技術仕様書

## 1. 技術スタック
- バックエンド: Python / Flask
- WSGI 実行: Gunicorn
- データベース: PostgreSQL
- LINE SDK: line-bot-sdk 3.x 系
- DB ドライバ: psycopg2-binary
- 時刻処理: 標準ライブラリの zoneinfo
- フロントエンド: HTML + CSS + JavaScript

## 2. 実行環境
- `Procfile` で `gunicorn main:app` を起動する。
- `runtime.txt` がある場合は対象 Python バージョンを固定する。
- 環境変数で機密情報と動作設定を与える。

## 4. アプリ構成
### 4.1 main.py
- Flask アプリ本体。
- ルーティング、DB 操作、LINE Webhook 処理、認証、セキュリティ制御を持つ。

### 4.2 templates/
- Jinja2 テンプレート。
- 管理画面、履歴、種類管理、ログイン、ログ閲覧を提供する。

### 4.3 static/
- `css/`: 画面スタイル
- `js/`: 画面操作、初期描画後の再取得、確認 UI、バージョン表示制御

## 5. DB 概要
### 5.1 reservations
- 予約本体
- 主な列: `id`, `user_id`, `message`, `status`, `type_id`, `created_at`, `called_at`, `arrived_at`, `completed_at`

### 5.2 reservation_types
- 予約種類
- 主な列: `id`, `name`, `owner_admin_id`, `accepting`, `created_at`

### 5.3 app_settings
- アプリ設定と自動呼出の履歴を保持
- 受付可否、自動呼出人数、前回・前々回の自動呼出結果などを保存

### 5.4 admin_login_logs
- 管理者ログイン履歴を保持

### 5.5 admin_accounts
- 管理者アカウント情報を保持
- 主な列: `id`, `login_id`, `password_hash`, `role`, `active`, `reservation_start_minute`, `reservation_end_minute`, `created_at`

### 5.6 login_attempt_records / webhook_request_records
- ログイン失敗回数と Webhook 受信回数のレート制限に使用する

## 6. 主な処理フロー
### 6.1 LINE 予約
1. Webhook 受信
2. 署名検証
3. メッセージ種別を確認（予約、キャンセル、到着、待ち時間）
4. 種類可否・受付時間・重複予約を判定
5. DB 更新
6. LINE へ返信

### 6.2 管理画面呼出
1. 管理者認証
2. 対象予約を取得
3. 状態変更
4. LINE 通知（失敗時はロールバック）
5. 画面反映

### 6.3 自動呼出タスク
1. バッチ認証トークン検証
2. 呼出タイムアウト対象を自動キャンセル
3. 5分単位で待機予約を自動呼出
4. 呼出結果と待ち時間目安を設定テーブルへ保存

### 6.4 管理画面表示更新
1. 初期描画はサーバーサイド HTML
2. 追加更新は `/admin/data` から取得
3. JS で DOM 差分更新

## 7. セキュリティ
- CSRF 対策
- セッション Cookie ハードニング
- Host 制限
- HTTPS 強制
- クリックジャッキング対策
- Webhook 署名検証
- ログインおよび Webhook のレート制限

## 8. パフォーマンス方針
- 管理画面の初期 JSON 埋め込みを避ける
- 必要なときだけ `/admin/data` を取得する
- JS の初期実行負荷を抑える
- DB 取得は必要列のみとし、履歴一覧はページングする
