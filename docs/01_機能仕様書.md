# 機能仕様書

## 1. システム概要
- 本システムは、LINE 公式アカウントを入口とした予約・呼出・到着受付・管理運用システムである。
- 一般利用者は LINE から予約、キャンセル、到着の操作を行う。
- 管理者は Web 管理画面から予約一覧の確認、呼出、到着完了、種類管理、履歴参照を行う。

## 2. 利用者種別
- 一般利用者: LINE で予約操作を行う。
- 管理者: 予約管理、種類管理、履歴閲覧、受付状態の制御を行う。
- 監査管理者: 管理ログを閲覧する。

## 3. 機能一覧
### 3.1 LINE 予約受付
- LINE のメッセージイベントを受信して予約を処理する。
- 利用者は「予約」「キャンセル」「到着」を送信できる。
- 予約時に種類名を指定できる。
- 種類未指定の場合は受付可能な種類の案内を返す。
- 種類が停止中の場合は予約を拒否する。

### 3.2 予約キャンセル
- 利用者が「キャンセル」を送信すると、直近の有効予約をキャンセルする。
- 対象がない場合はキャンセル不可メッセージを返す。

### 3.3 到着受付
- 利用者が「到着」を送信すると、直近の有効予約を到着済みに更新する。
- 呼出前の到着申請は拒否する。

### 3.4 管理画面ログイン
- 管理者はパスワードでログインする。
- 一般管理者と監査管理者を分けて扱う。
- ログイン失敗は記録される。

### 3.5 管理画面
- 現在の待機・呼出・到着予約を一覧表示する。
- 種類別の待機数を表示する。
- 新規受付の停止・再開を切り替えられる。
- 自動呼出人数を設定できる。
- 個別の呼出、到着完了処理を行える。

### 3.6 種類管理
- 予約種類を追加できる。
- 予約種類の受付可否を切り替えられる。
- 予約種類を削除できる。

### 3.7 履歴管理
- 完了・キャンセル・到着済みの予約履歴を表示する。
- 種類、状態、受付時刻、到着から完了時間で並べ替えできる。
- CSV で出力できる。

### 3.8 管理ログ閲覧
- 監査管理者はログイン履歴を閲覧できる。

### 3.9 自動呼出
- 5 分単位の実行タイミングで待機予約をまとめて呼び出す。
- 呼出結果は画面上の自動呼出サマリーに表示する。
- 実行記録は管理設定に保持する。

### 3.10 タイムアウト
- 呼出後 15 分で予約はタイムアウトし、自動でキャンセルされる。
- 呼出通知時に「15分以内」の案内とタイムアウト時刻を利用者へ通知する。
- タイムアウトで自動キャンセルされた場合は、その旨を利用者へ通知する。

## 4. 入力仕様
- 予約メッセージは最大文字数制限を持つ。
- 種類名は文字種と長さの制限を持つ。
- 管理画面の操作は CSRF トークンを必要とする。

## 5. 画面仕様
- ログイン画面
- 管理画面
- 種類管理画面
- 履歴画面
- ログ閲覧画面

## 6. 制約
- LINE Webhook は署名検証済みのリクエストのみ受け付ける。
- ログインと Webhook にはレート制限を適用する。
- セッションはアイドルタイムアウトする。
- HTTPS を基本とし、Host 制限を設定できる。
